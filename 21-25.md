Q: ทำไมต้องเปลี่ยนจาก impossible เป็น Low ?

A: ในระดับ Impossible โค้ดหลังบ้านได้รับการป้องกันอย่างสมบูรณ์ ทำให้ระบบไม่มีช่องโหว่และ sqlmap จะไม่สามารถเจาะระบบได้เลย เราจึงต้องลดระดับลงมาเป็น Low เพื่อจำลองระบบให้มีช่องโหว่ เพื่อเปิดโอกาสให้ sqlmap สามารถส่ง Payload เข้าไปแทรกคำสั่งและดึงข้อมูลออกมาให้เราได้

A VIP: ตอบแบบละเอียดขึ้นมานิดนึง โหมด Impossible คือ โหมดที่มีระดับปลอดภัยสูงสุด มีการใช้ Prepared Statements ทำให้ระบบทำการแยกคำสั่ง SQL ออกจากข้อมูลที่ผู้ใช้ป้อนอย่างเด็ดขาด ต่อให้เรายิง Payload ของ sqlmap หรือคำสั่งแฮกใด ๆ ลงไป ฐานข้อมูลจะมองสิ่งเหล่านั้นเป็นแค่ข้อความธรรมดา ๆ ตัวหนึ่ง ไม่นำไปประมวลผลเป็นคำสั่ง ทำให้ช่องโหว่ SQL Injection ไม่มีอยู่จริงในโหมดนี้ โปรแกรม sqlmap จึงไม่สามารถเจาะหรือดึงข้อมูลใด ๆ ออกมาได้ ส่วนโหมด Low คือโหมดที่มีระดับความปลอดภัยต่ำสุดหรือเป็นระดับเริ่มต้นสำหรับเรียนรู้ช่องโหว่นั่นเอง ไม่มีการกรองข้อมูลหรือการป้องกันใด ๆ ข้อมูลอะไรก็ตามที่ผู้ใช้พิมพ์เข้ามาในตัวแปร id จะถูกส่งวิ่งตรงไปประมวลผลที่ฐานข้อมูลทันที ทำให้ sqlmap สามารถส่งคำสั่ง SQL เข้าไปแทรกแซงโครงสร้างคำสั่งเดิมของระบบได้สำเร็จ

# Step 6 sqlmap

คำสั่งนี้เกรซเอามาจากคอร์ส Cisco เลย ปรับเปลี่ยนนิดหน่อยให้ตรงกับการทำโจทย์ของเรา

```
sqlmap -u "http://10.6.6.13/vulnerabilities/sqli/?id=1&Submit=Submit" --cookie="security=low; PHPSESSID=5ekgccs0ua74ri3e46ndcu6734" --dbs
```
อธิบายคำสั่ง

1. sqlmap เป็นการเรียกเปิดใช้งานโปรแกรม sqlmap
2. -u คือการระบุ URL เป้าหมาย ที่เราต้องการให้ sqlmap ตรวจสอบ
3. id=1&Submit=Submit คือการส่งพารามิเตอร์ผ่านแบบ GET Method สองตัวคือ id=1 และ Submit=Submit ตัวพารามิเตอร์ id=1 คือจุดที่ sqlmap จะพยายามส่ง Payload รูปแบบต่าง ๆ เข้าไปเพื่อทดสอบว่าเกิดการ Injection (แทรกคำสั่ง SQL) ได้หรือไม่
4. --cookie="security=low; PHPSESSID=5ekgccs0ua74ri3e46ndcu6734
   - --cookie ใช้สำหรับส่งค่า Session Cookie แนบไปกับการทดสอบ
   - security=low เป็นการกำหนดระดับความปลอดภัยของระบบ DVWA ในเซสชันนั้นให้เป็นระดับต่ำ เพื่อให้ง่ายต่อการทดสอบช่องโหว่
   - PHPSESSID ใช้ยืนยันตัวตนกับเว็บเซิร์ฟเวอร์
   เนื่องจาก DVWA บังคับให้เราล็อกอินก่อนถึงจะเข้าหน้าเว็บตามที่เราเห็นได้ ค่า PHPSESSID จึงทำหน้าที่เป็นเหมือนตั๋วผ่านทางให้ sqlmap สามารถวิ่งเข้าไปสแกนหลังบ้านได้ ส่วน security=low คือการบอกให้ระบบ DVWA ทำงานในโหมดป้องกันต่ำสุด เพื่อให้เราทดสอบช่องโหว่ได้ง่ายขึ้น
5. --dbs ย่อมาจาก Databases คำสั่งนี้จะบอกให้ sqlmap ไปทำการทดสอบหาช่องโหว่ SQL Injection บน URL นั้น เมื่อพบว่าช่องโหว่ใช้งานได้จริง มันจะยิงคำสั่งเพื่อไปดึงรายชื่อฐานข้อมูลทั้งหมด ที่ถูกจัดเก็บอยู่บน Database Server เครื่องนั้นออกมาแสดงผลบนหน้าจอของเรา

   เนื่องจาก DVWA บังคับให้เราล็อกอินก่อนถึงจะเข้าหน้าเว็บตามที่เราเห็นได้ ค่า PHPSESSID จึงทำหน้าที่เป็นเหมือนตั๋วผ่านทางให้ sqlmap สามารถวิ่งเข้าไปสแกนหลังบ้านได้ ส่วน security=low คือการบอกให้ระบบ DVWA ทำงานในโหมดป้องกันต่ำสุด เพื่อให้เราทดสอบช่องโหว่ได้ง่ายขึ้น
5. -D dvwa
   - -D ย่อมาจาก Database เป็นการระบุชื่อฐานข้อมูลที่ต้องการเจาะจงตรวจสอบ ในคำสั่งนี้คือการบอกให้โปรแกรมเข้าไปทำงานภายในฐานข้อมูลชื่อ dvwa
6. --dump-all เมื่อระบุคู่กับ -D dvwa คำสั่งนี้จะสั่งให้ sqlmap วิ่งเข้าไปในฐานข้อมูล dvwa แล้วทำการดึงข้อมูลจากทุกตารางและทุกคอลัมน์ที่อยู่ในฐานข้อมูลนั้นออกมาทั้งหมด มันจะดึงโครงสร้างตาราง ดึงรายชื่อฟิลด์ และดึงข้อมูลดิบทุกแถวที่ถูกบันทึกไว้ใน Database ออกมาแสดงผลบนหน้าจอ Terminal
